# F6-GATE-05E - Guía de comunicación y tono de soporte

## Objetivo

Crear una guía de comunicación para que el equipo DASC responda a clientes PyME con un tono claro, profesional y orientado a solución.

## Estado

En curso.

## Contexto

En F6-GATE-05A se definió el modelo de soporte sin GitHub.

En F6-GATE-05B se validó un ticket simulado.

En F6-GATE-05C se definieron canales reales y ticketing gratuito.

En F6-GATE-05D se crearon plantillas de respuesta.

F6-GATE-05E define el tono y las reglas de comunicación.


## Criterios principales

La comunicación debe ser:

- Clara.
- Profesional.
- Cercana.
- Sin exceso de tecnicismos.
- Sin culpar al cliente.
- Con próximos pasos concretos.
- Con evidencias al cerrar.

## Canales contemplados

~~~text

Formulario

Teléfono controlado
Jira
Zammad
Sistema interno de tickets
~~~

## Decisiones principales

El cliente no debe:

- Usar GitHub.
- Editar scripts.
- Tocar configuración interna.
- Ejecutar restauraciones sin acompañamiento.

El equipo DASC debe:

- Traducir problemas técnicos a lenguaje claro.
- Registrar internamente las solicitudes.
- Comunicar estado y próximos pasos.
- Cerrar con evidencia.

## Criterio de validación

F6-GATE-05E se considera preparada cuando:

- Existe una guía de tono y comunicación.
- Se definen frases recomendadas.
- Se definen frases a evitar.
- Se explica cómo comunicar incidencias técnicas.
- Se explica cómo comunicar restauraciones.
- Se mantiene GitHub fuera del soporte al cliente.
