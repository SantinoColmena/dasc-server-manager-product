# Validación R-032 - Modo demo sin datos sensibles

## Objetivo de la validación

Comprobar que la demostración de DASC Server Manager puede realizarse sin exponer información sensible ni ejecutar acciones peligrosas.

## Comprobaciones realizadas

| Comprobación | Resultado |
|---|---|
| No se muestran contraseñas reales | Pendiente |
| No se muestran tokens reales | Pendiente |
| No se muestran claves SSH privadas | Pendiente |
| Se usan datos de prueba | Pendiente |
| El usuario demo no tiene permisos de administración | Pendiente |
| Los backups usados son de laboratorio | Pendiente |
| Los logs no contienen datos sensibles | Pendiente |
| La demo puede repetirse sin romper el entorno | Pendiente |

## Evidencias recomendadas

Añadir capturas de:

1. Login con usuario demo.
2. Panel principal.
3. Vista de backups.
4. Backup de prueba ejecutado.
5. Vista de logs con evento registrado.
6. Vista de servicios.
7. Alertas o explicación del flujo.

## Resultado final

R-032 quedará cerrada cuando exista una demo segura, repetible y sin datos sensibles visibles.
