# F6-GATE-05D - Cierre plantillas de respuesta al cliente

## Objetivo

Cerrar la creación de plantillas de respuesta para que el equipo DASC pueda comunicarse con clientes PyME de forma clara, profesional y consistente.

## Estado

Cerrada.

## Contexto

En F6-GATE-05A se definió el modelo de soporte sin GitHub.

En F6-GATE-05B se validó el modelo mediante un ticket simulado.

En F6-GATE-05C se definieron canales reales y una capa opcional de ticketing gratuito.

F6-GATE-05D añade plantillas reutilizables para responder a clientes en distintos momentos del ciclo de soporte.

## Plantillas incluidas

| Plantilla | Uso |
|---|---|
| Confirmación de recepción | Avisar que la solicitud fue recibida |
| Solicitud de más información | Pedir datos necesarios para diagnosticar |
| Incidencia en análisis | Informar que el caso está siendo revisado |
| Servicio operativo | Explicar que no hay caída activa |
| Incidencia resuelta | Comunicar cierre correcto |
| Cambio planificado | Coordinar ventana de mantenimiento |
| Restauración de backup | Confirmar datos antes de restaurar |
| Incidencia crítica | Priorizar caída o problema grave |
| No reproducible | Cerrar caso sin fallo observable |
| WhatsApp breve | Confirmación rápida |
| Ticket interno creado | Informar referencia DASC |
| Mantenimiento programado | Avisar intervención planificada |

## Canales contemplados

Las plantillas pueden adaptarse a:

## Criterio de cierre

F6-GATE-05D se considera cerrada porque:

- Existen plantillas de respuesta al cliente.
- Las respuestas son claras y no excesivamente técnicas.
- Cubren recepción, análisis, cierre, restauración y mantenimiento.
- Sirven para canales simples y sistemas de ticketing.
- Mantienen GitHub fuera del canal de soporte al cliente.
- Permiten al equipo DASC responder de forma consistente.
