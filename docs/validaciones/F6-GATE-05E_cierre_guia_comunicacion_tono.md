# F6-GATE-05E - Cierre guía de comunicación y tono de soporte

## Objetivo

Cerrar la guía de comunicación y tono para soporte al cliente de DASC Server Manager.

## Estado

Cerrada.

~~~text
docs/soporte/guia_comunicacion_tono_soporte.md
~~~

## Validación creada


## Resultado

Se definió cómo debe comunicarse el equipo DASC con clientes PyME durante:

- Incidencias.
- Consultas.
- Restauraciones.
- Mantenimientos.
- Cambios planificados.
- Cierres de ticket.

## Criterios definidos

La comunicación debe ser:

- Clara.
- Profesional.
- Cercana.
- Sin exceso de tecnicismos.
- Sin culpar al cliente.
- Con próximos pasos concretos.
- Con evidencias al cerrar.

## Decisiones importantes

El cliente no debe:

- Usar GitHub.
- Editar scripts.
- Tocar configuración interna.
- Ejecutar restauraciones sin acompañamiento.

El equipo DASC debe:

- Traducir problemas técnicos a lenguaje claro.
- Registrar internamente las solicitudes.
- Comunicar estado y próximos pasos.
- Cerrar con evidencia.

## Criterio de cierre

F6-GATE-05E se considera cerrada porque:

- Existe guía de tono y comunicación.
- Existen frases recomendadas.
- Existen frases a evitar.
- Se explica cómo comunicar incidencias técnicas.
- Se explica cómo comunicar restauraciones.
- GitHub queda fuera del soporte al cliente.

## Conclusión

DASC Server Manager refuerza su soporte con una comunicación más profesional, clara y comprensible para clientes PyME.
