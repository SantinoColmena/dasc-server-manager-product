# R-047 - Release candidate v1.0-rc1

## Objetivo

Preparar una versión candidata de DASC Server Manager orientada a las primeras ventas y pilotos de pago.

Esta versión no se considera todavía una versión final de producción. Su objetivo es servir como base estable para instalar DASC en un primer cliente real o piloto de pago con alcance controlado.

## Estado de partida

La Fase 5 ha dejado validado el trabajo de pilotos reales:

- Piloto principal en perfil PyME.
- Piloto Lite con copia externa.
- Piloto 3 opcional documentado.
- Costes reales recalculados.
- SLA realista definido.
- Repositorio limpio antes de iniciar ventas.

## Alcance de v1.0-rc1

La versión candidata incluye:

- Instaladores del panel/API.
- Instalador de base de datos.
- Instalador de nodo de backups y servicios.
- Perfiles de despliegue Lite, PyME y Pro documentados.
- Backups completos, incrementales y diferenciales según el estado validado del proyecto.
- Restauración controlada.
- Historial de backups.
- Validaciones SHA256.
- Alertas básicas.
- Logs/auditoría.
- Documentación de instalación, soporte, SLA y costes.
- Checklist de instalación en cliente.

## Fuera de alcance

Esta release candidate no promete:

- Soporte ilimitado.
- Garantía absoluta de recuperación ante cualquier caso.
- Instalación sin validación previa del entorno.
- Uso en producción crítica sin revisión técnica.
- Sustitución completa de un proveedor de sistemas.
- Cumplimiento legal automático para cualquier sector.

## Criterios para considerar R-047 cerrada

R-047 se podrá marcar como hecha cuando:

- El repositorio esté limpio.
- Exista documentación de release candidate.
- Se haya revisado el README.
- Se haya creado una validación específica de R-047.
- Se haya definido qué entra y qué no entra en v1.0-rc1.
- Se haya creado un tag de release candidate.
- Exista un paquete o punto claro de instalación para cliente/piloto de pago.

## Nombre de versión propuesto

`v1.0-rc1`

## Decisión

La versión `v1.0-rc1` se usará como base comercial controlada para R-048, R-049 y R-050.

No se debe vender como versión final cerrada, sino como primera versión candidata acompañada de instalación, soporte limitado y validación del entorno.
