# Plantillas de respuesta al cliente - DASC Server Manager

## Objetivo

Definir respuestas base para atender solicitudes de soporte de clientes PyME de forma clara, profesional y consistente.

Estas plantillas pueden usarse por email, formulario, WhatsApp controlado o herramienta interna de ticketing.

## Principios de comunicación

Las respuestas al cliente deben ser:

- Claras.
- Breves.
- Educadas.
- Sin exceso de tecnicismos.
- Orientadas a solución.
- Con siguiente paso definido.
- Con evidencia cuando se cierre el caso.

## Plantilla 1 - Confirmación de recepción

### Uso

Cuando el cliente envía una incidencia, consulta o solicitud.

### Respuesta

~~~text
Hola,

Hemos recibido tu solicitud correctamente.

La hemos registrado internamente y vamos a revisarla según la prioridad indicada.

Te informaremos cuando tengamos un diagnóstico o si necesitamos más información.

Un saludo,
Equipo DASC
~~~

## Plantilla 2 - Solicitud de más información

### Uso

Cuando falta información para poder diagnosticar.

### Respuesta

~~~text
Hola,

Para poder revisar la incidencia necesitamos algunos datos adicionales:

- Servicio afectado.
- Fecha y hora aproximada del problema.
- Mensaje de error, si aparece.
- Captura de pantalla, si es posible.
- Indicar si el problema afecta a todos los usuarios o solo a uno.

Con esa información podremos continuar el diagnóstico.

Un saludo,
Equipo DASC
~~~

## Plantilla 3 - Incidencia en análisis

### Uso

Cuando el equipo DASC ya está investigando.

### Respuesta

~~~text
Hola,

Estamos revisando la incidencia reportada.

De momento el caso queda en análisis técnico. Estamos comprobando el estado de los servicios relacionados y revisando las evidencias disponibles.

Te informaremos con el resultado o con el siguiente paso necesario.

Un saludo,
Equipo DASC
~~~

## Plantilla 4 - Servicio operativo

### Uso

Cuando el cliente reporta posible caída, pero el servicio está funcionando.

### Respuesta

~~~text
Hola,

Hemos revisado el servicio y actualmente se encuentra operativo.

Las comprobaciones realizadas indican que el panel responde correctamente y que no hay una caída activa del servicio.

Si el problema continúa desde tu equipo o red, indícanos por favor:

- Hora exacta del intento.
- Usuario afectado.
- Captura del error.
- Navegador utilizado.

Un saludo,
Equipo DASC
~~~

## Plantilla 5 - Incidencia resuelta

### Uso

Cuando el problema ha sido corregido.

### Respuesta

~~~text
Hola,

La incidencia ha quedado resuelta.

Hemos aplicado las acciones necesarias y verificado que el servicio vuelve a funcionar correctamente.

Resumen:

- Servicio revisado.
- Corrección aplicada.
- Validación final realizada.

Damos el ticket por cerrado. Si vuelve a ocurrir, puedes responder a este mismo hilo o abrir una nueva solicitud.

Un saludo,
Equipo DASC
~~~

## Plantilla 6 - Cambio planificado

### Uso

Cuando una solicitud requiere una ventana de mantenimiento.

### Respuesta

~~~text
Hola,

La solicitud requiere una intervención planificada.

Para evitar impacto en el servicio, proponemos realizarla en una ventana de mantenimiento acordada.

Por favor, indícanos qué horario sería más adecuado para vuestra empresa.

Un saludo,
Equipo DASC
~~~

## Plantilla 7 - Restauración de backup solicitada

### Uso

Cuando el cliente solicita restaurar datos.

### Respuesta

~~~text
Hola,

Hemos recibido la solicitud de restauración.

Antes de ejecutar la restauración necesitamos confirmar:

- Base de datos o servicio afectado.
- Fecha aproximada a la que se desea restaurar.
- Motivo de la restauración.
- Confirmación de que entendéis que puede sobrescribir datos actuales.

Cuando tengamos esta confirmación, planificaremos la restauración y os informaremos del resultado.

Un saludo,
Equipo DASC
~~~

## Plantilla 8 - Incidencia crítica recibida

### Uso

Cuando el cliente reporta caída importante o pérdida de servicio.

### Respuesta

~~~text
Hola,

Hemos recibido la incidencia crítica.

Vamos a priorizar la revisión de este caso y comprobar el estado de los servicios principales.

Te informaremos lo antes posible con el diagnóstico inicial y las acciones necesarias.

Un saludo,
Equipo DASC
~~~

## Plantilla 9 - Cierre sin reproducción del problema

### Uso

Cuando no se consigue reproducir el fallo.

### Respuesta

~~~text
Hola,

Hemos revisado la incidencia, pero no hemos podido reproducir el problema en este momento.

Las comprobaciones actuales indican que el servicio responde correctamente.

Dejamos el caso cerrado como no reproducible. Si vuelve a ocurrir, por favor envíanos una captura y la hora exacta del fallo para poder investigarlo con más detalle.

Un saludo,
Equipo DASC
~~~

## Plantilla 10 - Confirmación por WhatsApp

### Uso

Respuesta breve para WhatsApp o canal rápido.

### Respuesta

~~~text
Hola, hemos recibido tu aviso.

Lo registramos internamente y lo revisamos. Te informaremos por este canal o por email cuando tengamos una actualización.
~~~

## Plantilla 11 - Respuesta de ticket creado en herramienta interna

### Uso

Cuando se usa Jira, Zammad u otra herramienta de ticketing.

### Respuesta

~~~text
Hola,

Hemos registrado tu solicitud en nuestro sistema interno de soporte.

Referencia interna:

DASC-YYYY-NNN

Usaremos esta referencia para hacer seguimiento del caso.

Un saludo,
Equipo DASC
~~~

## Plantilla 12 - Comunicación de mantenimiento

### Uso

Cuando se avisa al cliente de una intervención programada.

### Respuesta

~~~text
Hola,

Te informamos de que se realizará una intervención de mantenimiento programada.

Fecha:
Hora estimada:
Servicio afectado:
Impacto previsto:

Durante la intervención puede haber una interrupción temporal o degradación del servicio.

Te confirmaremos cuando el mantenimiento finalice.

Un saludo,
Equipo DASC
~~~

## Conclusión

Estas plantillas permiten responder de forma consistente sin obligar al cliente a entender detalles técnicos internos.

El equipo DASC puede adaptarlas según el canal, la prioridad y el tipo de solicitud.
